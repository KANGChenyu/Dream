# DreamLog 产品开发路线与 Web MVP 设计文档

## 目标

先把 `DreamLog-产品规划与技术架构.md` 里的完整产品路线落成可执行开发规划，同时明确第一阶段要做什么。

当前项目已经有产品规划文档和 FastAPI 后端骨架，但还没有用户真正能打开、能操作的前端应用。因此第一阶段先新增 `dreamlog-frontend/`，做出浏览器里可用的 Web MVP：用户可以登录、记录梦境、查看自己的梦境列表和详情。

视觉标准以根目录的 `Dream.png` 为准：暗色星空、紫蓝梦境光感、玻璃拟态卡片、中心沉浸式构图、两侧漂浮功能面板。整体要像一个 AI 梦境日志产品，而不是普通后台管理页面。

## 完整产品范围

完整 DreamLog 应覆盖产品规划文档中的能力：

- 用户系统：手机号登录、微信登录、JWT 鉴权、个人资料。
- 梦境记录：文字记录、日期、情绪、清晰度、清醒梦、公开/匿名设置。
- AI 解读：心理学、象征意义、文化视角、总结、建议、关键词。
- AI 绘梦：根据梦境描述生成梦境画面，支持不同风格。
- 分享卡片：生成可分享的梦境卡片，包含画面、摘要和传播入口。
- 我的梦境：列表、详情、搜索、时间线、日历视图。
- 梦境标签：AI 自动提取关键词，支持筛选。
- 社区广场：公开梦境流、热门排序、最新排序、瀑布流/卡片展示。
- 社区互动：点赞、评论、回复、通知。
- 撞梦匹配：基于 pgvector 的语义相似度检索，展示匹配梦境和原因。
- 语音记录：语音转文字，并可由 AI 优化梦境描述。
- 梦境统计：高频元素、情绪趋势、做梦频率。
- 梦境连续剧：识别连续梦境并生成剧情线。
- 梦境地图：全站热门梦境元素可视化。
- 专业解读：心理咨询师或专家付费深度解读。
- 会员体系：会员权益、高级绘梦风格、无限解读次数、支付。
- 梦境周报：每周生成个人梦境分析报告。
- 多端：Web 端、小程序端，后续可扩展 H5。
- 增长与运营：SEO、分享裂变、每日话题、推荐、内容审核、举报、运营后台。

## 分阶段路线

### Phase 1：Web MVP 记梦工具

目标：先让项目从“只有接口文档”变成“有真实产品界面”。

范围：

- Web 手机号验证码登录。
- 梦境记录表单。
- 我的梦境列表。
- 梦境详情。
- `Dream.png` 风格视觉外壳。
- AI 解读/绘梦入口预留。

里程碑：用户可以在浏览器里登录、写梦、回看梦境。

### Phase 2：AI 解读与绘梦闭环

目标：实现产品的核心吸引力。

范围：

- 后端补齐 AI 解读任务流。
- 前端详情页展示心理学、象征意义、文化视角、总结和建议。
- 生成关键词标签。
- 接入 AI 绘梦，生成梦境画面。
- 如果真实 API Key 不稳定，先提供 mock AI provider，保证开发可测。

里程碑：用户记录梦境后，可以获得 AI 解读和梦境画面。

### Phase 3：分享与个人管理

目标：让用户愿意保存和分享。

范围：

- 梦境分享卡片。
- 标签筛选。
- 我的梦境搜索。
- 时间线视图。
- 日历视图。
- 基础情绪趋势。

里程碑：个人梦境管理体验完整，并具备传播素材。

### Phase 4：社区与互动

目标：从工具变成社区。

范围：

- 社区广场。
- 公开梦境 Feed。
- 热门/最新排序。
- 点赞。
- 评论和回复。
- 通知。
- 内容举报和基础审核。

里程碑：用户能浏览他人的公开梦境并互动。

### Phase 5：撞梦匹配

目标：形成 DreamLog 的差异化社交体验。

范围：

- 生成梦境 embedding。
- pgvector 相似度检索。
- 撞梦结果展示。
- AI 生成匹配原因。
- 从详情页引导用户查看“和你做了相似梦的人”。

里程碑：用户能发现和自己梦境相似的匿名用户。

### Phase 6：小程序端

目标：适配更适合传播和日常记录的端。

范围：

- uni-app 或 Taro 小程序初始化。
- 微信登录。
- 小程序端记录、列表、详情、分享。
- 与 Web 共用后端接口。

里程碑：小程序端具备核心使用体验和微信分享能力。

### Phase 7：高级功能与商业化

目标：提升留存、增长和收入能力。

范围：

- 语音记录。
- 梦境统计。
- 梦境周报。
- 梦境连续剧。
- 梦境地图。
- 专业解读。
- 会员体系和支付。
- SEO、运营工具、推荐策略。

里程碑：产品具备长期运营和商业化基础。

## 第一版范围

第一版先做“个人记梦工具”闭环：

- 手机号验证码登录，使用现有后端的开发模式短信验证码流程。
- 梦境记录表单：梦境内容、日期、情绪、清晰度、是否清醒梦、是否公开、是否匿名。
- 我的梦境列表：展示日期、情绪、标题或内容摘要、公开/私密状态。
- 梦境详情页：展示完整梦境内容和元数据。
- AI 解读和 AI 绘梦区域先作为“即将接入”或禁用入口展示，保留视觉位置，但不阻塞第一版可用性。

第一版暂不做：

- 社区广场。
- 分享卡片生成。
- 真实 AI 解读或 AI 绘图。
- 微信小程序。
- 支付、会员、报告、统计、梦境地图等高级功能。

## 技术方向

在项目根目录新增 `dreamlog-frontend/`，使用：

- Vite
- React
- TypeScript

前端调用现有 FastAPI 后端，默认后端地址为 `http://localhost:8000`。

需要接入的接口：

- `POST /api/v1/auth/sms/send`
- `POST /api/v1/auth/login/phone`
- `GET /api/v1/auth/me`
- `POST /api/v1/dreams`
- `GET /api/v1/dreams`
- `GET /api/v1/dreams/{dream_id}`
- `PUT /api/v1/dreams/{dream_id}`
- `DELETE /api/v1/dreams/{dream_id}`

MVP 阶段将 access token 存到 localStorage。前端封装一个轻量 API client，统一附加 `Authorization: Bearer <token>`。

初始路由：

- `/login`：手机号验证码登录页。
- `/`：主工作台，记录梦境 + 我的梦境列表。
- `/dreams/:id`：梦境详情页。

## 视觉系统

界面要贴近 `Dream.png`：

- 全屏暗色宇宙/梦境背景，带紫蓝光晕。
- 首屏有明显的 DreamLog / AI 梦境日志气质。
- 使用半透明玻璃面板、细边框、模糊、柔光、轻微发光效果。
- 中心区域承载“记录梦境”的核心交互。
- 侧边面板承载梦境列表、关键词、AI 解读入口、情绪洞察等信息。
- 移动端自动堆叠，不能出现文字重叠或面板挤压。

不要做成白底 SaaS 工具页。不要只做营销落地页。第一屏必须能直接开始记录梦境。

## 数据流程

登录流程：

1. 用户输入手机号。
2. 前端调用 `/auth/sms/send`。
3. 开发环境后端返回 `debug_code`。
4. 前端允许用户填写验证码；如果接口返回了 `debug_code`，可以辅助显示或自动填入。
5. 前端调用 `/auth/login/phone`。
6. 保存 token 和用户信息，进入 `/` 主工作台。

创建梦境流程：

1. 用户填写梦境内容和相关字段。
2. 前端校验梦境内容长度和日期。
3. 前端调用 `POST /dreams`。
4. 创建成功后刷新梦境列表，并展示刚创建的梦境。

列表和详情流程：

1. 主工作台加载时调用 `GET /dreams`。
2. 用户点击某条梦境，进入 `/dreams/:id`。
3. 详情页调用 `GET /dreams/{id}` 获取完整数据。

## 错误处理

- 手机号、验证码、梦境内容、日期都要有表单校验。
- 登录失败时保留用户输入，并显示简短错误提示。
- 后端无法连接时，展示明确的“后端服务未连接”提示。
- token 失效时清空登录状态并回到登录页。
- 梦境列表为空时，展示鼓励用户写下第一个梦的空状态。

## 后端前提

后端可能还需要为了前端 MVP 做少量补齐：

- 本地依赖需要安装，或者使用 Docker 运行。
- 数据库表需要能创建。
- CORS 已包含 `http://localhost:5173`。
- 开发环境短信接口会返回 `debug_code`。

如果后端暂时不能持久化数据，实现阶段应补上最小可用的数据库初始化或迁移方案，确保 Web MVP 能跑通。

## 验证标准

完成后需要验证：

- 前端可以在 `http://localhost:5173` 启动。
- 手机号验证码登录能连接本地后端。
- 创建梦境后能持久化。
- 我的梦境列表能显示刚创建的梦。
- 梦境详情页能显示正确数据。
- 桌面端和移动端浏览器检查无文字重叠、无面板错位、无明显视觉破碎。

## 实施计划预览

1. 确认后端本地运行方式和 API 可用性。
2. 创建 `dreamlog-frontend/`，初始化 Vite + React + TypeScript。
3. 实现 API client 和登录状态管理。
4. 实现符合 `Dream.png` 氛围的暗色梦境视觉外壳。
5. 实现手机号验证码登录。
6. 实现梦境记录表单、我的梦境列表、梦境详情页。
7. 补齐 loading、empty、error、unauthorized 状态。
8. 用浏览器验证桌面端和移动端布局，并修正响应式问题。
